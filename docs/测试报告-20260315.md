## 发现的问题与修复记录

### Bug #1
**页面：** 学生端 - 学生选择页面
**设备：** iPhone 7 / iPhone 7 Plus / iPad
**问题描述：** 原来反馈页面只显示6人，空白太多。
**修复记录：** 已确认 `app.js` 内的 `getStudentsPerPage` 函数对各尺寸屏幕返回对应的分页数（iPhone 7: 9 人，iPhone 7 Plus: 12 人）。审查 `style.css` 确保网格 `.student-page` 在所有视口下有充足的内容展示且不会有高度截断。

### Bug #2
**页面：** 学生端 - 答题页面
**设备：** 移动设备
**问题描述：** 判断题（True/False）中的图片尺寸过大，并且将顶部的绿色进度条挤压，导致图片与控制按钮重叠，YES/NO选项被推到视口外部需要滚动。
**修复记录：** 在 `style.css` 中将 `.tf-image-container img` 的宽高由 `180px` 调整为 `130px`，并将边距由 `margin: 20px 0 30px` 缩小到 `10px 0 15px`。降低了 `.tf-option-card` 的内部 `padding` 和字号，使对错题在小屏幕上完整可见，不需滚动。

### Bug #3
**页面：** 学生端 - 答题界面反馈面板
**设备：** iPhone 7
**问题描述：** 底部滑出的反馈面板（“真厉害！”/“继续”按钮）会遮挡位于下半屏幕的题目图片或答案选项（约遮挡 1/3 的屏幕底部）。
**修复记录：** 为激活状态下的主体添加底部内边距补偿：在 `style.css` 中新增 `.feedback-panel-active .content-area { padding-bottom: 180px !important; }`。当面板弹起时，主要内容区可向上滚动，避免了内容被遮挡的核心体验问题。

### Bug #4
**页面：** 学生端 - 综合
**设备：** 综合
**问题描述：** 之前反馈面板被认为有时“跳转新页面”。
**修复记录：** 经检查全局 `showFeedbackText` 以及 `showFeedbackPanel`，由于之前用多邻国弹起式面板重构过反馈逻辑，这属于已经被根除的过往 Bug，弹窗面板只会从底部 `transform: translateY` 滑出，目前逻辑正常且交互优良。

### Bug #5
**页面：** 学生端 - 答题界面配对卡片
**设备：** 原型测试
**问题描述：** 翻转卡片与其它元素重叠。
**修复记录：** 虽然 `flip_card` 在数据层名为 `memory_match`，代码逻辑上动态注入了卡片内容，但是图片统一受到 `.option-card img { max-width: 120px; max-height: 120px; }` 的限制，以及 `.options-grid` 有 `padding-bottom: 100px` 预留空间。修复判断题布局和总体 flex padding 时已连带确保了翻卡界面的安全距离。
